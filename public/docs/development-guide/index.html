如果你的目标是开发或维护一个真实机器人插件，建议先看 ReKep 样例插件页；如果你的目标是把某个硬件快速接到 HAL 中，建议先看 Driver 接入页；如果你已经在改代码并希望保持项目的一致性，则应先阅读代码风格与命名约定页。

开发子页面 以 ReKep 为样例的插件开发指南 解释插件目录、清单文件、runtime 分层、本地部署、预检与 dry-run，以及如何把 ReKep 的结构迁移到新插件。 开发子页面 接入新 Driver 的完整流程 从 `BaseDriver` 的最小接口开始，覆盖 profile、动作映射、运行态回写、Watchdog 验证与全链路联调。 开发子页面 代码风格、命名约定与目录约定 整理主仓库与插件仓库中更适合沿用的代码组织方式，包括命名、目录边界、职责拆分与文档约定。

这部分文档会解决什么问题

如何开始

帮助第一次进入项目的开发者确认：先装什么、先看什么、怎样跑通最小闭环，以及哪些现象可以作为系统正常运行的基线。

如何扩展

覆盖插件开发、Driver 接入与 runtime 分层的基本做法，避免把主仓库、插件逻辑和第三方 SDK 混在一起。

如何维持一致性

补充命名约定、目录边界、代码风格和测试建议，帮助新增代码更自然地融入现有仓库，而不是成为额外的分支风格。

推荐的阅读路径

如果你要开发新插件

先看 ReKep 样例插件页，再回到 Driver 接入页补齐统一执行接口的理解。这样更容易区分“插件内部 runtime 逻辑”和“PhyAgentOS 侧 Driver 适配”这两层职责。

如果你要接一个新本体

先看 Driver 接入页，确认最小接口和联调路径；再看代码风格页，避免新代码在命名、目录结构和职责边界上偏离主仓库习惯。
