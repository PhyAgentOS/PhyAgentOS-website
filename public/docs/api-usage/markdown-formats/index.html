API Usage / Markdown Protocol

Markdown 文件格式与样例

这一页聚焦工作区中的协议文件本身：哪些文件属于 shared workspace，哪些属于 robot workspace；每类文件主要由谁写入、被谁读取；以及这些文件在运行时通常应当长成什么样。它更适合在排查状态异常、确认字段含义或编写兼容工具时参考。

1. 先区分 shared workspace 和 robot workspace

在阅读这些文件之前，先区分它们出现在哪个工作区非常重要。single 模式下所有文件可能都在同一个目录里；fleet 模式下，shared workspace 和每台机器人的 workspace 会分开。

位置 典型文件 主要用途
shared workspace ENVIRONMENT.md、ROBOTS.md、LESSONS.md、TASK.md、ORCHESTRATOR.md 保存全局状态、共享环境、机器人摘要和任务协调信息
robot workspace ACTION.md、EMBODIED.md 保存单个机器人实例自己的动作队列和运行时能力画像

2. ENVIRONMENT.md：全局环境真相源

这个文件是当前环境状态的主承载面。它通常由 HAL Watchdog 和感知 / 导航相关模块更新，Agent 和 Critic 会从这里读取场景图、机器人运行态和地图信息。

{
  "schema_version": "PhyAgentOS.environment.v1",
  "updated_at": "2026-03-17T10:20:30Z",
  "scene_graph": {
    "nodes": [
      {
        "id": "obj_red_apple",
        "class": "apple",
        "object_key": "red_apple",
        "center": {"x": 0.05, "y": 0.05, "z": 0.75},
        "size": {"x": 0.06, "y": 0.06, "z": 0.06},
        "confidence": 0.96,
        "frame": "map",
        "track_id": "track_apple_001",

      }
    ],
    "edges": [
      {"source": "obj_red_apple", "relation": "ON", "target": "furniture_table", "confidence": 0.97}
    ]
  },
  "robots": {
    "go2_edu_001": {
      "connection_state": {
        "status": "connected",
        "transport": "ssh",
        "host": "192.168.1.23",
        "port": 22,
        "last_heartbeat": "2026-03-17T10:20:30Z"

这里最容易混淆的有两点：一是 `scene_graph` 和 `objects` 可以同时存在；二是运行时连接状态和导航状态应该写在 `robots.` 下，而不应该塞进 `EMBODIED.md`。

3. EMBODIED.md：单个机器人实例的能力画像

`EMBODIED.md` 不是实时状态文件，它更像一份给 Critic 和 Agent 阅读的能力说明书。运行时通常由 Watchdog 从 `hal/profiles/*.md` 复制到对应 robot workspace。

# EMBODIED.md Template

## Identity
- **Name**: Unitree Go2 EDU
- **Type**: quadruped
- **Driver/Profile**: go2_edu

## Sensors
- **Camera**: RGBD
- **LiDAR**: yes
- **Odometry**: yes

## Supported Actions
| Action | Parameters | Description |
|--------|-----------|-------------|
| `move_to` | `x, y` | Navigate to a target pose |
| `target_navigation` | `target_label, timeout_s` | Move toward a visual target |

## Physical Constraints
- **Max speed**: 1.0 m/s
- **Safety distance**: 0.4 m
- **Collision policy**: stop when obstacle too close

写这个文件时，重点不是把所有技术细节塞满，而是让 Agent 足够明确地知道：这台机器人是什么、它能做什么、什么不能做、哪些动作需要额外谨慎。

4. ACTION.md：单个机器人自己的动作队列

`ACTION.md` 是执行链路中最直接的动作派发文件。对单个机器人来说，它只服务于该实例自己的 Watchdog。

```json
{
  "schema_version": "PhyAgentOS.action_queue.v1",
  "actions": [
    {
      "id": "2f18c6a912ab",
      "action_type": "target_navigation",
      "parameters": {
        "robot_id": "go2_edu_001",
        "target_label": "apple",
        "timeout_s": 30
      },
      "status": "pending"
    }
  ]
}
```
字段 说明
schema_version当前动作队列协议版本
actions动作数组，Watchdog 会寻找第一个 pending 项
id动作唯一 ID
action_type动作类型，例如 `target_navigation`、`pick_up`、`real_execute`
parameters动作参数对象
status`pending` / `completed` / `failed` / `cancelled`

5. LESSONS.md 与 ROBOTS.md：一个记录失败，一个提供摘要

这两个文件经常被忽略，但对理解系统实际行为非常有帮助。

LESSONS.md

# Lessons Learned

## Failed Action Attempt
- **Action**: pick_up
- **Parameters**: {"target": "red_apple"}
- **Reasoning**: The apple is on the table and should be reachable.
- **Critic Rejection**: target not confirmed in current environment state

ROBOTS.md

# Robot Registry

| Robot ID | Driver | Type | Capabilities | Workspace | Enabled | Profile | Connection | Navigation |
|---|---|---|---|---|---|---|---|---|
| go2_edu_001 | go2_edu | quadruped | move_to, target_navigation, ... | `~/.PhyAgentOS/workspaces/go2_edu_001` | yes | go2_edu.md | connected | running |

6. 谁读取什么：最实用的心智模型

角色 主要读取 主要写入
主 Agent ENVIRONMENT.md、ROBOTS.md、LESSONS.md、TASK.md、ORCHESTRATOR.md 动作草案、任务拆解、消息输出
Critic / EmbodiedActionTool shared `ENVIRONMENT.md`、目标机器人 `EMBODIED.md`、当前动作草案 ACTION.md 或 LESSONS.md
Watchdog robot workspace `ACTION.md`、shared `ENVIRONMENT.md` runtime `EMBODIED.md`、更新后的 `ENVIRONMENT.md`、`ROBOTS.md`
如果排查时不知道先看哪个文件，一个简单判断是：先看当前这个问题更像“规划问题”“能力约束问题”还是“执行回写问题”，再对应到上面的读取关系去定位文件。
这一页更适合作为运行时协议参考页使用：当你想确认某个 Markdown 文件应该长什么样、由谁维护、在什么阶段被读取时，可以优先回到这里。
