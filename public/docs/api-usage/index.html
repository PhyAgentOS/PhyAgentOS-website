总览：建议的阅读顺序

如果你的目标是“先把系统用起来”，建议先看插件使用；如果你的目标是“弄清工作区里每个 Markdown 文件到底应该长什么样”，建议先看文件格式与样例页。两者分别服务于不同阶段：前者帮助你跑通执行链路，后者帮助你判断运行状态是否合理。

API 子页面 插件使用说明 聚焦插件的安装、注册、调用路径、ReKep 的使用方式，以及调试 runtime 时的常见入口。 API 子页面 Markdown 文件格式与样例 逐个解释 `ENVIRONMENT.md`、`EMBODIED.md`、`ACTION.md`、`LESSONS.md`、`ROBOTS.md` 等文件的格式、字段与示例。

按目标选择阅读路径

如果你是使用者或演示操作者

建议先看插件使用说明页，重点关注插件安装、Watchdog 启动方式、`ACTION.md` 直接调用和 runtime 调试命令。这样更容易先把执行链路跑起来。

如果你是开发者或排查者

建议先看 Markdown 文件格式与样例页。理解 `ENVIRONMENT.md`、`EMBODIED.md` 与 `ACTION.md` 的结构之后，再回到插件页看具体的调用方式，会更容易定位问题。
